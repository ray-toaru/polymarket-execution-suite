# Security Policy

Report vulnerabilities privately through GitHub's private vulnerability
reporting feature when available. If that feature is unavailable, contact the
repository owner directly through a private channel.

Do not open a public issue containing private keys, API credentials, account
identifiers, raw signatures, signed payloads, or exploitable production
details.

Include the affected commit, impact, reproduction steps using synthetic data,
and a proposed containment action. This repository does not promise live or
production readiness; a security report must not be treated as authorization
to exercise real-funds paths.
