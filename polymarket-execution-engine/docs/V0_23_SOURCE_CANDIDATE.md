# v0.23.0 source candidate

> Status: current v0.23.0 source-candidate documentation. Historical gate-specific notes are archived under `docs/archive/`; current validation entrypoint is `validation/run_current_gates.sh`.

## Decision

Promote the previous v0.22 + v0.23-continuation patch into a coherent `0.23.0` source candidate, while explicitly avoiding a validated-release claim.

## Evidence

Confirmed in source:

- Root `VERSION` is `0.23.0`.
- Hermes package metadata and `__version__` are `0.23.0`.
- Rust workspace and official SDK adapter manifests are `0.23.0`.
- Release manifest is `0.23.0` and says source-candidate/not-production.
- CI uses `validation/run_current_gates.sh`.
- Local path package entries in Cargo.lock have been aligned to `0.23.0` where manifests require it.

not confirmed here:

- `cargo check --workspace --locked`.
- `cargo clippy`.
- Rust tests.
- PostgreSQL migration/store/API E2E.
- SDK adapter/spike tests.
- Credentialed non-trading smoke.
- Sign-only dry-run with real credentials.

## Rationale

Keeping the package named v0.22 while adding v0.23 APIs created audit and release confusion. Directly calling it a validated v0.23 release would also be inaccurate because Rust/PG gates were not run here. The minimal sufficient position is therefore `v0.23.0 source candidate`.

## Attack / defense review

### Round 1

- Current conclusion: bump to v0.23.0 source candidate.
- Defense: source has v0.23 scope and stale metadata was confirmed.
- Attack: bumping without Rust gates may overstate confidence.
- Accepted attack: yes, if described as a validated release.
- Revision: use source-candidate/not-production status and require external gates.

### Round 2

- Current conclusion: align Cargo.lock local package version entries.
- Defense: path package manifest versions changed; lockfile mismatch was a known `--locked` risk.
- Attack: lockfiles should normally be generated by Cargo, not hand-edited.
- Accepted attack: partially. This environment lacks Cargo, so only version metadata was aligned; the dependency graph was not changed.
- Revision: require `cargo check --workspace --locked` as the source of truth.
