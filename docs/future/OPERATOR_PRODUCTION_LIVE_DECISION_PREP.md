## Required operator inputs

The operator decision cannot be completed until the current packet includes all
of the following, bound to the exact artifact and evidence manifest under
review:

- current release artifact, SHA-256 sidecar, evidence sidecar, provenance, and
  workspace manifest;
- current full gate evidence, including Rust, PostgreSQL, SDK adapter,
  credentialed non-trading, sign-only dry-run, store-truth/readback, release
  hygiene, docs/evidence governance, contract validation, and CI evidence;
- operator approval request with `status=operator_approval_request_not_authorization`,
  `live_submit_authorized=false`, and `remote_side_effects_authorized=false`;
- candidate market file hash, condition id, whitelist/caps review, account
  binding, and runtime truth hash;
- external references for secret custody, operator approval, alert routing,
  dashboard, rollback runbook, incident runbook, reconcile health, cancel-only
  fallback, and kill switch;
- independent dual-control review whose reviewer identity differs from the
  operator identity and whose evidence contains no secrets, raw signatures, raw
  signed payloads, or signed order envelopes.

## Guarded generation path

If and only if those inputs are complete and current, the reviewed-go release
decision must be generated by the guarded tooling rather than hand-edited:

```bash
cd polymarket-execution-engine
python validation/prepare_reviewed_go_decision.py \
  --approval-request-file <approval-request.json> \
  --external-references-file <external-references.json> \
  --dual-control-review-file <dual-control-review.json> \
  --decision-id <operator-reviewed-go-id> \
  --decision-reason <operator-reviewed-reason> \
  --output <release-decision.json>
```

That script validates the approval request, external references, dual-control
review, hash bindings, runtime gate snapshot, and final release-decision schema.
Running it would create an authorizing `reviewed_go` decision, so it must not be
run from this prep document alone.
